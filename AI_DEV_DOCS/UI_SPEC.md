# UI/UX Specification

## Design Direction

Based on the inspiration images, Happenlist's design should feel:

- **Warm and approachable** â€” cream/beige backgrounds, soft greens, warm accents
- **Clean and uncluttered** â€” generous whitespace, clear hierarchy
- **Modern but not cold** â€” rounded corners, subtle shadows, friendly typography
- **Mobile-first** â€” touch-friendly targets, easy scrolling

### Inspiration Notes

- **Travelly (Image 1)**: Warm cream background, green accents, clear filters sidebar, card-based listings
- **Event Ticket UI (Image 2)**: Bold date display, category chips, dark header with location
- **Event Booking (Image 3)**: Hero images, price badges, organizer profiles, map integration
- **Ride App (Image 4)**: Illustrated style, playful elements, card selection patterns

## Interaction States

### Buttons

- **Default**: Background color, no shadow
- **Hover**: Slightly darker, subtle shadow
- **Active/Pressed**: Darker, pressed appearance
- **Disabled**: Muted colors, no pointer
- **Loading**: Spinner, disabled state

### Cards

- **Default**: Subtle shadow
- **Hover**: Lift effect (translate-y, shadow increase)
- **Focus**: Visible outline for keyboard nav

### Form Inputs

- **Default**: Light border, white background
- **Focus**: Primary color border, subtle glow
- **Error**: Red border, error message below
- **Disabled**: Gray background, muted text
